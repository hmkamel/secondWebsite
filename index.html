The purpose of this website is to document my journey in learning about Web Development as well as serve as practice as I learn. Also it'll be interesting to see how much I've developed as a aspiring web developer.

  • It is important to keep your lines of code structured in order for it to be more legible.
  • Such as to have everything included in your head & body to be indented.
  • Best practices is to replace italics with emphasis as well as replace bold with strong.
  • Using alt after inserting an image URL can be good for search enging optimization because Google uses alt texts to find out what a webpage is all about.
  • Anchors can be used as such.
  • When creating Forms on HTML I would not recommend to put it into a table format.
  • Browser have a default value for css & html elements. If your css style isn't functioning as intented try seeing what its default value is.
  • You can create classes amongest your elements and use .class caller in the external css files to individualy manipulate them. Also you can have multiple classes per element.
  • You can create ids as well as classes with id="" and call that in the css external styler with #. However with id's they are unique and can only be used on one single element
  • Elements are or boxes are shaped by the following: its dimension, its border dimensions, its padding (space between the box element and the border), and margin dimensions(how far away are two elements from each other)
  • CSS Display properties: have something called block elements where they block any elements from being next to them either to the right or left they are; paragraphs, headers, divisions, lists, and forms.
  • Span can be used with paragraphs to just edit a few letters of the whole paragraphs and they are also knows as inline display elements.
  • You can change the display properties of elements from block to inline or even a mix of both as inline-block
  • Positioning can be a property that can be changed - like using position: relative; where you can move the element however many pixels you want from its natural position
  • Divs from what I understand so far are used to position elements where you want them to be in a page. Those elements positions can be changed relative to those divs or can be fixed in place using "fixed" to persist throughout the page even when you're scrolling.
  • Using rem for font size means root em so it will only take that size into consideration regardless of parent attributes.
  • You can use the float property for an image to be wrapped around in another elements like a paragraph for example.
  • You can use clear for that paragraph to not appear either to the right of left of a floating image to give it its own line space.
